The Problem

Highway incidents go undetected
for critical minutes

Oregon DOT operates hundreds of TripCheck CCTV cameras along the state highway system. Currently, operators must manually watch these feeds to spot crashes — a task that is slow, error-prone, and impossible to scale across an entire network.

Every minute between a crash and an emergency response increases the risk of secondary incidents, traffic backup, and fatalities. Automated detection changes that calculus entirely.

Our system continuously polls TripCheck camera images, runs YOLO-based inference on each frame, and writes crash-classified images to disk organized by severity level — giving operators and downstream alerting tools an automatic signal instead of a manual one.

Key Features

What the system does

Real-Time YOLO Inference

A custom-trained YOLO model classifies each TripCheck camera still image as crash or no_crash, with severity-tier labeling. Tuned for edge-device performance on Raspberry Pi hardware.

TripCheck API Integration

Authenticates with ODOT's TripCheck CCTV Inventory API to retrieve live camera lists across multiple regions, deduplicates by device ID, and downloads still images on a configurable polling cycle.

Severity Classification

Detected crashes are automatically sorted into severity-tiered output folders. Severity thresholds and folder mappings are fully configurable via config.yaml, making the system adaptable to operator workflows.

Raspberry Pi Deployment

Designed from the ground up for edge deployment on a Raspberry Pi 5. No cloud dependency, no GPU required — the full pipeline runs offline on local hardware, within the Pi's 4 GB RAM envelope.

Configurable & Maintainable

All thresholds, API credentials, timing, and paths live in config.yaml and .env. Model weights are swappable — retraining produces a new .pt file with no code changes needed.

25,000-Image Training Set

Trained on a growing dataset of real highway crash scenes — labeled with rotated bounding boxes via Roboflow, augmented for varied lighting and camera angles. Continuously expanded to close detection gaps.

How It Works

The detection pipeline

01 — FETCH
Camera Inventory
tripcheck_fetcher.py calls the ODOT TripCheck API with a regional bounds parameter to retrieve the full list of active CCTV cameras and their current still-image URLs.
02 — DOWNLOAD
Still Image Pull
For each camera, the fetcher performs an HTTPS GET for the current still frame. Failed downloads are logged and skipped — the cycle continues uninterrupted.
03 — INFER
YOLO Detection
crash_detector.py loads local YOLO weights and runs inference on the frame. The model outputs a classification: no_crash or a severity label.
04 — SAVE
Severity Routing
Crash-classified images are written to the corresponding severity output folder. Clean frames are discarded, keeping storage usage minimal and output immediately actionable.
05 — LOOP
Continuous Polling
The orchestrator in main.py repeats the cycle on a configurable interval — refreshing the camera list and processing every active feed, indefinitely.

Run it yourself
